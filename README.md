# 🤖 بوت تيليجرام لتحويل المشاريع إلى APK

هذا البوت يقوم باستقبال ملفات المشاريع البرمجية (Flutter, React Native, Android Native, Web) أو روابط مستودعات GitHub ويقوم ببنائها وتحويلها إلى ملف APK جاهز باستخدام **GitHub Actions**.

## 🚀 المميزات
- **مجاني تماماً:** يعتمد على أدوات مجانية (GitHub Actions, Render.com).
- **مستقل:** يعمل بشكل تلقائي بمجرد إعداده ولا يحتاج لاشتراكات مدفوعة.
- **آمن:** مخصص للاستخدام من قبل الأدمن فقط.
- **مرونة في الإدخال:** يدعم رفع الملفات المضغوطة (ZIP) وروابط مستودعات GitHub.
- **تفاعل محسّن:** رسائل واضحة ومفصلة عن حالة عملية البناء.

---

## 🛠️ متطلبات التشغيل
1. **توكن بوت تيليجرام:** احصل عليه من [@BotFather](https://t.me/BotFather).
2. **GitHub Personal Access Token:**
   - اذهب إلى إعدادات حسابك في GitHub.
   - Developer Settings -> Personal Access Tokens -> Tokens (classic).
   - قم بإنشاء توكن جديد بصلاحيات `repo` و `workflow` و `delete_repo`.
3. **حساب على Render.com:** لاستضافة البوت مجاناً.

---

## 📦 كيفية الإعداد والتشغيل

### 1. إعداد المتغيرات البيئية
قم بإنشاء ملف `.env` ووضع البيانات التالية فيه. هذه المتغيرات ضرورية لعمل البوت بشكل صحيح، خاصة عند الرفع على Render.com:
```env
BOT_TOKEN=توكن_البوت_الخاص_بك
ADMIN_ID=معرف_التليجرام_الخاص_بك
GITHUB_TOKEN=توكن_جيت_هاب
GITHUB_USER=اسم_مستخدم_جيت_هاب
```

### 2. التشغيل محلياً (للتجربة)
```bash
pip install -r requirements.txt
python bot.py
```

### 3. الرفع على Render.com (للتشغيل الدائم)
لضمان عمل البوت بشكل مستقل ودائم، اتبع الخطوات التالية لرفعه على Render.com:
1.  **إنشاء مستودع GitHub:** قم بإنشاء مستودع (Repository) جديد على حسابك في GitHub وارفع عليه جميع ملفات البوت (`bot.py`, `workflows.py`, `requirements.txt`, `README.md`, `.env.example`).
2.  **ربط Render.com بـ GitHub:** في Render.com، اختر **New Web Service**، ثم اربط حسابك في GitHub واختر المستودع الذي رفعته.
3.  **تكوين الإعدادات:**
    -   **Runtime:** Python
    -   **Build Command:** `pip install -r requirements.txt`
    -   **Start Command:** `python bot.py`
4.  **إضافة المتغيرات البيئية:** أضف المتغيرات البيئية (Environment Variables) في إعدادات Render كما هي في ملف `.env` (BOT_TOKEN, ADMIN_ID, GITHUB_TOKEN, GITHUB_USER). تأكد من أن هذه المتغيرات مضبوطة بشكل صحيح لضمان عمل البوت.
5.  **النشر:** قم بنشر الخدمة. سيبدأ Render.com في بناء وتشغيل البوت تلقائياً.

---

## 🏗️ كيف يعمل البوت؟
1.  يرسل المستخدم ملف `.zip` للمشروع أو رابط مستودع GitHub.
2.  يقوم البوت بفحص الملف/الرابط وتحديد نوع المشروع (Flutter, React Native, Android Native, Web).
3.  يتم إنشاء مستودع GitHub مؤقت جديد ورفع ملفات المشروع إليه، بالإضافة إلى ملف GitHub Actions Workflow المناسب.
4.  يتم تفعيل GitHub Actions لبدء عملية بناء التطبيق.
5.  بعد الانتهاء من البناء، يقوم البوت بتحميل ملف APK الناتج وإرساله للمستخدم.
6.  يقوم البوت بحذف المستودع المؤقت وجميع الملفات لتنظيف المساحة والحفاظ على الخصوصية.

## 📝 ملاحظات هامة
-   الحد الأقصى لحجم الملف هو 50 ميجابايت (قيود تيليجرام).
-   عملية البناء قد تستغرق من 5 إلى 15 دقيقة حسب نوع المشروع وحجمه.
-   تأكد من أن مشروع Flutter أو React Native يحتوي على جميع الملفات الأساسية في الجذر (Root) لكي يتم التعرف عليه بشكل صحيح.
-   لضمان استمرارية عمل البوت، تأكد من أن حسابك على Render.com يستخدم الخطة المجانية التي تدعم التشغيل الدائم (Always On) إذا كانت متاحة، أو قم بمراقبة استهلاك الموارد.
